flask command relies on FLASK_APP environment variable to know the location of the application
flask run - sub command native to Flask to run the application
flask db - to manage everything related to database migrations
flask db init - create a migration repository
flask db migrate -m "users table" - add the User model to the migration script/generates the migration script. This is executed everythime there is a change in the database schema/application models
The command output will tell where it wrote the migration script
To make changes to the db, 
flask db upgrade
To undo the last migration, 
flask db downgrade
